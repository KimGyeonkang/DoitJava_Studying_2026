
## 자바 입출력 기능이란?
우리가 흔히 사용하는 카카오톡 같은 메신저나 인스타그램, X 등 SNS, 내가 쓰고 있는 티스토리(블로그) 등 **각종 파일(자료)를 올리고 저장하고 쓰고 이를 프로그램에서 출력하는 모든 기능**

## 스트림(Stream)
- 자바에서 모든 입출력을 관장하는 통로
- 자바에서는 입출력 장치와 무관하게 일관성 있는 스트림을 제공한다.
- 자료 입출력을 위한 모든 기능을 모아둔 집합(클래스)를 **스트림 클래스**라고 한다.
*- (주의) 자바에서 자료 처리 기능을 구현한 클래스인 스트림(Stream)과는 다른 개념!*

## 스트림에는 어떤 종류가 있을까?

1. 표준 입출력
- 표준 입출력 클래스: 자바에서 화면에 출력과 입력을 담당

| 스트림 이름(자료형) | 변수명 | 사용 예시 |
|-------|-------|-------|
| static PrintStream  | out  | System.out(표준 출력 스트림) |
| static InputStream  | in | System.in(화면에서 값을 입력받는 표준 입력 스트림)   |
| static OutputStream  | err   | System.err(빨간색 오류 메시지 출력)   |

-> 표준 입출력 클래스 변수는 모두 _static(정적) 변수_

### Scanner 클래스
- java.util에 있는 입력 클래스
- 문자, 정수, 실수 등 자료형과 파일, 문자열 등 여러 대상으로부터 자료를 읽어들임.

(사용 예) Scanner scanner = new Scanner(System.in)
표준 입력 스트림인 System.in을 매개변수로 Scanner 생성자를 실행해, 새로운 클래스를 생성하고 값을 입력 받음.

2. 입력, 출력 스트림
- 어떤 대상으로 자료를 읽어들임 -> 입력 스트림
- 사용자가 편집 창에서 쓴 글을 파일에 저장 -> 출력 스트림
- 입력과 출력은 한 스트림에서 동시에 일어날 수 없다.
- 대개 스트림명에 Input이나 Reader가 포함되면 입력 스트림, Output이나 Writer가 포함되면 출력 스트림

3. 단위 스트림
- 바이트 단위: 자바의 기본 자료 입출력 방식(Byte 단위)
-> 다만 흔히 사용하는 한글 같은 문자는 바이트 단위로 정상 출력하기 어렵다. 문자 하나 즉 char형 데이터는 2바이트 크기를 갖기 때문이다. 이런 한계를 보완하고자 문자 단위 스트림을 사용한다.
- 문자 단위: 프로그램에서 가장 많이 사용하는 문자 형식 데이터를 읽어들이기 위한 방식
* 이름이 Stream으로 끝나면 바이트 단위 스트림, Reader나 Writer로 끝나면 문자 단위 스트림

4. 기반과 보조
- 기반 스트림: 자료를 직접 읽거나 쓰는 기능
- 보조 스트림: 직접 자료를 읽고 쓸 수는 없지만 타 스트림에 부가 기능을 제공
* 보조 스트림은 다른 말로 **Wrapper** 스트림이라고도 함.
-> 자체적인 입출력 기능은 없으나 자신이 **감싸고 있는** 기반 스트림이 기능을 수행할 때 보조 기능을 추가
- 기반 스트림을 중심으로 다수의 보조 스트림을 추가해 기능 확장 가능
-> 데코레이터(Decorator): **기능을 유동적이고 효율적으로 추가하거나 제거**할 수 있는 디자인 패턴(구조 패턴의 일종). 자바의 입출력 스트림 클래스가 대표적(실제로 역할하는 기반 스트림에 추가 기능을 제공하는 보조 스트림 추가)
- 기반 스트림: FileInputStream, FileOutputStream(이상 모든 보조 스트림의 상위 클래스), FileReader, FileWriter 등
- 보조 스트림: InputStreamReader, OutputStreamWriter(이상 생성된 바이트 스트림->문자로 변환), BufferedInputStream, BufferedOutputStream(이상 버퍼링 기능: 배열 제공으로 더 빠른 입출력 실행 보조, 한 바이트나 문자 단위 처리보다 더 빠른 속도) 등.*

5. 직렬화와 역직렬화
- 직렬화: 인스턴스(객체)의 특정 지점을 게임 테이터마냥 세이브하거나 네트워크를 통해 전송하는 기능\
-> 객체 내용(변숫값)을 연속 스트림으로 만들어 파일로 저장하거나 네트워크로 전송\
-> 반대로 저장되었거나 전송받은 내용을 다시 돌려놓는 기능은 역직렬화

6. 그 외 어떤 입출력 클래스가 있을까?
- File 클래스: '파일' 개념을 추상화한 클래스로\
따로 입출력 기능은 없으나, 파일 자체의 경로나 정보를 알아내거나 파일 생성 가능
- RandomAccessFile 클래스: 랜덤한 위치의 자료를 읽음.\
읽어들인 파일의 어느 위치에서 읽고 쓰는지 가리키는 **파일 포인터**가 중요
